## 2024-05-18 - Fix CSV Injection and structural breakage
**Vulnerability:** A local structural and CSV injection vulnerability was found in the application's file storage implementation (`FilePersistenceService.java`). Commas and newlines in user-supplied strings directly break the custom CSV-parsing logic (`String.split(",")`), crashing the file loading or resulting in unexpected structures. Further, formulas beginning with `=`, `+`, `-`, or `@` could be exploited by external spreadsheet programs if the raw CSVs are ever exported.
**Learning:** Writing custom CSV serializers using simple string concatenation logic almost always leads to structure breakage or injections. When native CSV libraries are unavailable, a manual sanitation utility is essential.
**Prevention:** To prevent CSV structure breakage, replace user-supplied commas, carriage returns, and newlines with spaces. To prevent CSV Injection, unconditionally prefix fields beginning with `=`, `+`, `-`, or `@` with a single-quote (`'`).
