## 2026-05-06 - Prevent Sensitive Information Exposure in Error Messages\n**Vulnerability:** Raw exception details (`e.getMessage()`) were being directly printed to console output and standard error.\n**Learning:** In a CLI environment, raw exception messages can easily leak internal system paths, configuration states, or unhandled file conditions to the user.\n**Prevention:** Always use generic, sanitized error messages (e.g., "An unexpected error occurred") when catching exceptions that are surfaced to users.
