## 2024-05-10 - Clear Calls to Action for Empty States
**Learning:** Terminal CLI apps suffer from dead-end states when users encounter empty data (like no budgets or transactions). A simple text output saying "No budgets defined" leaves the user unsure of next steps, degrading the UX.
**Action:** Always provide explicit, numbered options as calls to action for empty states. E.g., when no budgets exist, guide the user directly to the menu option to add one: "No budgets defined. Use option [4] from the menu to set one."
