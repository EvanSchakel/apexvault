## 2026-05-06 - Actionable Empty States in CLI Tables
**Learning:** Empty tables or simple 'No data' messages are a missed opportunity to guide the user. Giving specific next steps in empty states improves the CLI experience significantly.
**Action:** Always provide an actionable empty state (e.g., 'No budgets defined. Use option 4 to set a category budget.') instead of just rendering an empty table or a generic message.
